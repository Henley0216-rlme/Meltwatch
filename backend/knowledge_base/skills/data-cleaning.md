# 数据清洗 Skill

## 目标

读取一份评论汇总 CSV 文件（1000+ 条），严格按照 Meltwatch 的「从评论到分析维度」管线，将非结构化评论文本转化为可量化、可聚合的维度标签集，输出为 `维度标签集.md`。

## 核心原则

1. **每一步标注数据形态变化**：明确输入是什么、输出是什么
2. **保留原始评论**：任何标签/维度结论必须可下钻回溯到原文
3. **标注置信度**：对于 LLM 推断的字段（场景、人群），标注推断置信度
4. **统计与个体并存**：输出既要有个体评论的标签向量，也要有聚合统计

---

## 第一步：数据导入与概览

### 操作
- 读取用户指定的 CSV 文件（默认编码 UTF-8，备选 GBK）
- 自动识别列名，映射到标准字段
- 输出数据概览：总评论数、各平台分布、时间跨度、品类分布

### 标准字段映射
CSV 中的列名可能是中文或英文，尝试自动映射：
- `平台` / `platform` / `source`
- `品类` / `category` / `product_type`
- `时间` / `时间戳` / `timestamp` / `date` / `time`
- `评论正文` / `评论` / `内容` / `text` / `review` / `content`
- `评分` / `rating` / `score`（可选，如无则标为"未知"）
- `产品名称` / `product` / `product_name`（可选）

### 输出（此时不做修改，仅记录）
```
## 数据概览
- 总评论数：XXX
- 平台分布：京东 XX条, 淘宝 XX条, ...
- 时间跨度：YYYY-MM-DD ~ YYYY-MM-DD
- 品类分布：...
- 存在评分的记录：XX%
```

## 第二步：数据清洗

严格按照以下顺序执行，每一项标注执行结果。

### 2.1 去重
- 完全重复的行 → 删除，保留首次出现
- 同一用户 + 近似相同内容（编辑距离 > 0.9）→ 标记为疑似重复，降权
- 统计：去除了多少条，剩余多少条

### 2.2 去噪
筛选并标记以下低质量评论（标记为 `质量标记: 噪音`，不直接删除，供后续参考）：
- 纯数字/纯符号/纯表情
- 长度 < 4 个字符且无语义内容（如"好评""666""。"）
- 明显的广告内容（含 URL、推广链接、重复模板话术）
- 与品类无关的灌水内容
- 统计：标记为噪音的数量

### 2.3 格式标准化
统一以下字段：
- **平台**：统一为 `京东` / `淘宝` / `小红书` / `抖音` / `大众点评` / `微博` / `拼多多` / `其他`
- **时间**：统一为 `YYYY-MM-DD` 格式
- **评分**：统一为 1-5 分制（如原始为 10 分制则除以 2，如原始为百分制则除以 20）
- **文本**：去除首尾空白、统一全角半角标点

### 2.4 元数据补全
- 识别时间是否处于特殊事件窗口（大促/节假日/新品发布季）：
  - 618（5月底-6月中）、双11（10月底-11月中）、双12、年货节、春节、国庆等
  - 标注到 `事件窗口` 字段
- 从评论内容推断品类（如果 CSV 中未提供）

### 清洗输出
完成去重+去噪+标准化+补全后，输出清洗报告：
```
## 清洗报告
- 原始总数：XXX
- 去重后：XXX（去除 XX 条）
- 噪音标记：XX 条
- 有效评论：XXX 条
- 格式标准化：时间 XX 条修正, 评分 XX 条修正
```
清洗后的数据保留在上下文中，进入下一步。

## 第三步：结构化提取

这是核心步骤。对每条有效评论进行三层处理。

### 3.1 关键词体系（三层提取）

对每条评论，分别提取三类关键词，并标注该词的情感极性（正/中/负）：

**产品特征词**
识别用户关注的产品维度，将提取的词归入以下类别之一：

| 类别 | 示例关键词 |
|------|-----------|
| 质量 | 做工、材质、耐用、结实、掉色、坏了、粗糙 |
| 外观 | 好看、颜值、颜色、设计、款式、丑、老气 |
| 功能 | 好用、实用、效果、性能、功能、方便 |
| 性价比 | 便宜、贵、划算、值得、不值、性价比 |
| 包装 | 包装、盒子、精美、简陋、破损 |
| 尺寸/规格 | 大小、尺码、容量、重量、厚度 |

**场景词**
识别使用/购买场景，归入以下类别之一：

| 类别 | 示例关键词/表达 |
|------|---------------|
| 自用 | 自己用、买给自己、日常、通勤、上班 |
| 送礼 | 送人、礼物、送礼、给XX买的、包装要好看 |
| 居家 | 家里用、放家里、居家、家居 |
| 户外/旅行 | 出门、旅行、出差、户外、携带 |
| 季节-冬季 | 冬天、保暖、暖和、防寒、羽绒 |
| 季节-夏季 | 夏天、凉快、透气、清爽、防晒 |

**人群词**
推断用户画像 / 送礼对象，归入以下类别之一：

| 类别 | 示例关键词/表达 |
|------|---------------|
| 学生 | 学生、上学、宿舍、考研、性价比高 |
| 上班族 | 上班、通勤、办公室、开会、出差 |
| 宝妈 | 宝宝、孩子、婴儿、小孩、安全 |
| 银发族 | 老人、长辈、爸妈、爷爷、奶奶、操作简单 |
| 送礼-送长辈 | 给爸妈买的、送长辈、老人用、父母 |
| 送礼-送伴侣 | 男朋友、女朋友、老公、老婆、对象 |
| 送礼-送孩子 | 给孩子买的、小朋友、儿子、女儿 |
| 价格敏感 | 便宜、性价比、太贵了、不值这个价 |
| 品质导向 | 质量好、大牌、贵点没关系、一分钱一分货 |

**提取原则**：
- 每条评论可以属于多个类别
- 先做关键词匹配（基于上述示例词 + 近义词扩展）
- 对于关键词匹配不到但上下文暗示的场景/人群 → 用 LLM 语义推断，标注 `置信度: 推断`
- 关键词匹配到的标注 `置信度: 匹配`

### 3.2 权重计算与模糊归一

**3.2.1 提及率权重（全局统计）**
- 统计每个维度类别在所有评论中的提及次数
- 计算提及率 = 提及该维度的评论数 / 总有效评论数
- 输出各维度的提及率排名

**3.2.2 情感交叉权重**
- 对每条评论，结合情感极性标注，计算每个维度的情感得分：
  - 正面提及 = +1，负面提及 = -1，中性提及 = 0
  - 该维度的情感得分 = (正面数 - 负面数) / 提及总数
  - 范围：-1.0 到 +1.0

**3.2.3 时间衰减权重**
- 以最新评论日期为基准
- 发布时间距今越近权重越高
- 简单方案：最近30天权重1.0，30-90天权重0.8，90-180天权重0.5，180天以上权重0.3

**3.2.4 模糊归一（同义词映射）**
对以下同义表达进行归一合并：
- 物流相关：快递/物流/配送/发货/到货/等了X天/还没到 → 统一归入「服务-物流」
- 质量相关：质量/做工/材质/品质/做工 → 统一归入「产品-质量」
- 价格相关：价格/价钱/贵/便宜/性价比/划算 → 统一归入「产品-性价比」
- 客服相关：客服/售后/态度/回复/投诉 → 统一归入「服务-客服」
- 外观相关：外观/颜值/好看/设计/款式/颜色 → 统一归入「产品-外观」

### 3.3 维度标签向量（每条评论的输出格式）

对每条有效评论，输出以下结构化记录：

```markdown
| ID | 平台 | 品类 | 时间 | 事件窗口 | 场景标签 | 人群标签 | 维度情感 | 情感总分 | 质量标记 | 原文(截断) |
|----|------|------|------|---------|---------|---------|---------|---------|---------|-----------|
| 001 | 京东 | 羽绒服 | 2025-11-15 | 双11 | 自用,冬季保暖 | 上班族(推断) | 质量+1,物流-1,外观+1 | +0.67 | 有效 | 穿了一周很暖和，就是快递太慢... |
```

字段说明：
- `场景标签`：逗号分隔的场景类别
- `人群标签`：逗号分隔，括号标注置信度（匹配/推断）
- `维度情感`：逗号分隔的 `维度:分数`，未提及的维度不列出
- `情感总分`：所有提及维度的平均情感得分
- `质量标记`：有效/噪音

## 第四步：生成维度标签集

将第三步的输出汇总为 `维度标签集.md`，包含以下部分：

### 4.1 数据概览（来自第一步）
### 4.2 清洗报告（来自第二步）
### 4.3 维度提及率总览

按维度的聚合统计表：

```markdown
| 维度类别 | 具体维度 | 提及次数 | 提及率 | 正面占比 | 负面占比 | 情感得分 | 趋势信号 |
|---------|---------|---------|--------|---------|---------|---------|---------|
| 产品 | 质量 | 245 | 24.5% | 68% | 22% | +0.46 | — |
| 产品 | 外观 | 189 | 18.9% | 80% | 8% | +0.72 | — |
| 服务 | 物流 | 156 | 15.6% | 35% | 55% | -0.20 | ⚠ |
| ... | ... | ... | ... | ... | ... | ... | ... |
```

趋势信号：
- ⚠ 标记：负面占比 > 40% 且提及率 > 10%
- ↑ 标记：近30天提及率较前30天上升 > 30%
- ↓ 标记：近30天提及率较前30天下降 > 30%

### 4.4 场景 × 维度交叉分析

```markdown
| 场景 | Top1 关注维度 | Top2 关注维度 | Top3 关注维度 | 代表性评论摘要 |
|------|-------------|-------------|-------------|-------------|
| 送礼 | 包装(+0.8) | 外观(+0.7) | 质量(+0.5) | "送人的包装要好看" |
| 自用 | 性价比(+0.3) | 功能(+0.6) | 质量(+0.4) | ... |
| 冬季保暖 | 保暖效果(+0.8) | 质量(+0.5) | 物流(-0.3) | ... |
```

### 4.5 人群 × 场景交叉洞察

```markdown
| 人群 | 主要场景 | 核心关注维度 | 痛点维度 | 情感倾向 |
|------|---------|-------------|---------|---------|
| 宝妈 | 送礼-送孩子 | 安全性,材质 | — | 偏正面 |
| 上班族 | 自用-通勤 | 便携性,外观 | 物流 | 中性 |
| ... | ... | ... | ... | ... |
```

### 4.6 典型评论样本

每个维度取正面/负面各 3 条最具代表性的评论（代表性强 = 维度关键词密度高 + 观点清晰）：

```markdown
### 质量维度 - 正面代表
- [京东/2025-11-15] "做工很精细，用料扎实，穿了一个月没变形"（质量+1,外观+1）
- ...

### 质量维度 - 负面代表
- [淘宝/2025-10-20] "穿了一周就开线了，质量太差了"（质量-1）
- ...
```

### 4.7 数据完整度说明
- 标注哪些字段是爬取获取的、哪些是 AI 推断的
- 标注数据中无法获取的字段
- 评估整体数据可信度

## 注意事项

1. **全程保留原文**：每条评论的处理结果必须附带原文（可截断至60字），确保可回溯
2. **大文件处理**：如果 CSV 超过 2000 条，先输出概览统计，再分批处理，每批 500 条
3. **编码兼容**：CSV 文件可能使用 UTF-8 或 GBK 编码，读取出错时尝试切换编码
4. **中途保存**：每处理完一批（500条），保存中间结果到 `维度标签集_进度.md`，防止上下文丢失
5. **最终输出文件名**：`维度标签集.md`

- [ ] 每条有效评论都有维度标签向量
