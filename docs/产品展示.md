```

│   "差评过了 3 天才发现，已经发酵成投诉了"                     │

## 02 解决方案

### ReviewPulse 是什么

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   🔍 输入        →    🤖 AI 分析      →    📊 输出         │
│                                                             │
│   大众点评评论        正面/负面判断        可视化看板        │
│   淘宝评价            情感打分             差评预警          │
│   京东评论            关键词提取           运营建议          │
│   拼多多             问题归类             趋势图表          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 核心功能

| 功能 | 说明 | 客户价值 |
|------|------|----------|
| **情感分析** | AI 自动判断正面/负面 | 5 秒内完成 1000 条分析 |
| **多平台采集** | 统一采集各平台评论 | 一个后台看全平台评价 |
| **差评预警** | 实时通知负面评价 | 第一时间处理，避免发酵 |
| **趋势洞察** | 情感变化趋势分析 | 提前发现产品/服务问题 |
| **运营建议** | 根据分析结果推荐行动 | 告诉商家"该怎么做" |

### 客户案例（虚构示例）

| 商家 | 痛点 | 使用后效果 |
|------|------|-----------|
| 某服装旗舰店 | 日均 2000 条评论，人工看不过来 | AI 自动分类，客服效率提升 80% |
| 某火锅连锁 | 差评发现晚，口碑受损 | 差评预警，响应时间从 3 天→2 小时 |
| 某电器品牌 | 不知道用户不满的原因 | 关键词提取，发现"快递损坏"是主因 |

## 04 技术架构

### 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **AI 模型** | RoBERTa (大众点评数据微调) | 二分类，正面/负面 |
| **后端** | Flask + Python | 轻量级 API 服务 |
| **数据库** | PostgreSQL + SQLite | 用户数据 + 分析记录 |
| **前端** | 原生 HTML/CSS/JS | 无框架依赖 |
| **部署** | Docker + Nginx | 一键部署 |

### 数据流向

```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│ 用户输入  │ →  │  API 网关 │ →  │ AI 推理  │ →  │ 数据存储 │
│ /爬虫    │     │  (Nginx) │     │ (PyTorch)│     │ (SQL)   │
└──────────┘     └──────────┘     └──────────┘     └──────────┘
                                            │
                                            ▼
                                     ┌──────────┐
                                     │ 前端展示 │
                                     │ Dashboard │
                                     └──────────┘
```

### 核心 API

| 接口 | 说明 |
|------|------|
| `POST /api/v1/analyze` | 单条评论情感分析 |
| `POST /api/v1/batch_analyze` | 批量分析（最多 20 条） |
| `GET /api/v1/crawl/platforms` | 获取支持的爬虫平台 |
| `POST /api/v1/crawl/scrape` | 爬取指定 URL 的评论 |

| 中国电商商家总数 | 2,000 万+ |
| 中大型商家（年销 100 万+） | ~100 万 |

| 可达市场（SAM） | ¥10 亿/年 |
| **3 年目标份额** | **1% = ¥1,000 万/年** |

### 市场趋势

```
✅ 电商持续增长（15 万亿市场规模）
✅ 商家精细化运营需求上升
✅ AI 成本大幅下降（本地模型无需 API 费用）
✅ 暂无绝对领先者（市场分散）
```

### 切入路径

```
第1步：中小商家（低门槛，快速验证）
    ↓
第2步：连锁品牌（多店铺管理，痛点强）
    ↓
